# Helper function
def print_metrics(name, y_true, y_pred, y_train_true=None, y_train_pred=None):
    print(f"--- {name} ---")
    if y_train_true is not None:
        print(f"  Train R²:   {r2_score(y_train_true, y_train_pred):.4f}")
        print(f"  Train RMSE: {np.sqrt(mean_squared_error(y_train_true, y_train_pred)):.4f}")
    print(f"  Test R²:    {r2_score(y_true, y_pred):.4f}")
    print(f"  Test RMSE:  {np.sqrt(mean_squared_error(y_true, y_pred)):.4f}")
    print(f"  Test MAE:   {mean_absolute_error(y_true, y_pred):.4f}")

a) Ordinary Least Squares (OLS) Regression¶

OLS minimizes the sum of squared residuals without any regularization:

$$\hat{\boldsymbol{\beta}}_{OLS} = \arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|^2$$

The closed-form solution is:

$$\hat{\boldsymbol{\beta}}_{OLS} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$$

b) Ridge Regression (L2 Regularization)¶

Ridge regression adds an L2 penalty to the OLS objective:

$$\hat{\boldsymbol{\beta}}_{Ridge} = \arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|^2 + \alpha \|\boldsymbol{\beta}\|_2^2$$

The closed-form solution is:

$$\hat{\boldsymbol{\beta}}_{Ridge} = (\mathbf{X}^T\mathbf{X} + \alpha \mathbf{I})^{-1}\mathbf{X}^T\mathbf{y}$$

Key properties:

  • Shrinks all coefficients toward zero but never exactly to zero
  • Handles multicollinearity by adding $\alpha \mathbf{I}$ to the Gram matrix
  • The regularization parameter $\alpha$ controls the trade-off between fit and complexity

c) LASSO Regression (L1 Regularization)¶

What is LASSO?¶

LASSO (Least Absolute Shrinkage and Selection Operator) adds an L1 penalty:

$$\hat{\boldsymbol{\beta}}_{LASSO} = \arg\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|^2 + \alpha \sum_{j=1}^{p} |\beta_j|$$

Key properties:

  • The L1 penalty creates a non-differentiable objective at $\beta_j = 0$
  • This causes some coefficients to become exactly zero, performing automatic feature selection
  • Produces sparse models that are easier to interpret
  • Particularly useful when only a subset of features are truly relevant
  • Unlike Ridge, there is no closed-form solution; it is solved via coordinate descent or subgradient methods
  • From a convex optimization perspective, the L1 penalty is convex but not strictly convex, which leads to the sparsity-inducing property

Discussion¶

Effect of Regularization on Coefficients¶

  1. OLS provides the unbiased maximum likelihood estimates. All coefficients are freely estimated, which can lead to overfitting when features are correlated or when the model is too flexible relative to the sample size.

  2. Ridge (L2) shrinks all coefficients toward zero proportionally. No coefficient is set exactly to zero — all features remain in the model. This is beneficial when we believe all features contribute to the prediction and we want to reduce variance without eliminating predictors. The L2 penalty also stabilizes the solution when features are multicollinear (the $(\mathbf{X}^T\mathbf{X} + \alpha\mathbf{I})$ matrix is always invertible for $\alpha > 0$).

  3. LASSO (L1) can shrink coefficients exactly to zero, effectively removing features from the model. This performs automatic feature selection and produces a sparse, more interpretable model. The features that survive LASSO's penalty are those that the algorithm considers most relevant for prediction.

Regularization Parameter Selection¶

  • Both Ridge and LASSO optimal $\alpha$ values were selected via 5-fold cross-validation over a logarithmically-spaced grid. This ensures the choice is driven by out-of-sample generalization performance rather than in-sample fit.

Recommendation¶

Examine the results table above:

  • If all three models achieve similar test performance, LASSO may be preferred for its interpretability and feature selection properties.
  • If Ridge achieves noticeably better CV performance, it suggests that all features contribute meaningfully and regularization without elimination is beneficial.
  • The gap between train and test R² indicates overfitting; smaller gaps suggest better generalization.

In general, for a small dataset (100 observations, 6 features), regularization helps prevent overfitting, and cross-validation is essential to obtain reliable estimates of model performance.
